XPath Stability & AutoHeal Test

Test XPath stability with changing button labels and AutoHeal with randomized attributes on every visit.

XPath Stable Button

The button text changes on every page load / refresh, but the absolute XPath (/html/body/main/div[2]/div[1]/div/button) remains the same. Use this to verify that XPath-based selectors still locate the element even when visible text changes.

How it works: On every page load or refresh, the button text is randomly picked from a pool of labels (e.g., "Submit", "Proceed", "Continue", "Go", "Execute", "Confirm", "Send", "Apply", "Save", "Next"). The id, class, and DOM position never change — only the visible text and data-label attribute change. The absolute XPath always resolves to the same element.

AutoHeal Test Elements

These elements have fixed visual positions and purpose, but every attribute (id, class, name, data-*, aria-label, placeholder text) is randomized on each page load. Any XPath or CSS selector from a previous visit will break, forcing AutoHeal to engage.
